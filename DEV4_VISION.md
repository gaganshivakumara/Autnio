## Vision Pipeline

### How it works

The phone camera (accessed via browser `getUserMedia`) captures frames in the web app. The app uploads each frame to a pre-signed S3 URL, then calls the vision Lambda endpoint. The Lambda preprocesses the image and invokes the appropriate Bedrock model.

```
Phone camera (getUserMedia in browser)
     │
     ▼
Web app captures frame as JPEG blob
     │
     ▼
Upload to pre-signed S3 URL (generated by a lightweight Lambda)
     │  S3 key passed to vision router
     ▼
vision/router Lambda
     │
     ├── mode=detect  →  vision/qwen-detect Lambda  →  Qwen3-VL-235B
     └── mode=stream  →  vision/nemotron-stream Lambda  →  Nemotron Nano 2 VL
     │
     ▼
Result (description / bounding boxes) returned to web app
Web app reads result aloud via browser TTS (SpeechSynthesis API)
```

### Routing rules

```python
# functions/vision/router.py
def handler(event, context):
    body = json.loads(event['body'])
    mode = body.get('mode', 'detect')   # default to high-accuracy detection

    if mode == 'stream':
        return invoke_nemotron(body['imageS3Key'], body.get('prompt', 'Describe the scene'))
    else:
        return invoke_qwen(body['imageS3Key'], body.get('prompt', 'Detect and locate all objects'))
```

## Vision Lambda Functions

### `functions/vision/preprocessor.py`

Resizes and tiles input images to 512×512 before sending to Bedrock (both models expect this format).

```python
import boto3, base64
from PIL import Image
import io

s3 = boto3.client('s3')

def load_and_encode(bucket: str, key: str) -> str:
    obj = s3.get_object(Bucket=bucket, Key=key)
    img = Image.open(io.BytesIO(obj['Body'].read())).convert('RGB')
    img = img.resize((512, 512))
    buf = io.BytesIO()
    img.save(buf, format='JPEG')
    return base64.b64encode(buf.getvalue()).decode('utf-8')
```

### `functions/vision/qwen-detect.py`

Invokes `qwen.qwen3-vl-235b-a22b` for on-demand object detection and grounding.

```python
import boto3, json, os
from preprocessor import load_and_encode

bedrock = boto3.client('bedrock-runtime')
MODEL_ID = os.environ['QWEN_VL_MODEL_ID']   # qwen.qwen3-vl-235b-a22b

def invoke_qwen(s3_key: str, prompt: str) -> dict:
    image_b64 = load_and_encode(os.environ['S3_BUCKET'], s3_key)

    body = {
        "messages": [{
            "role": "user",
            "content": [
                {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{image_b64}"}},
                {"type": "text", "text": prompt}
            ]
        }],
        "max_tokens": 1024
    }

    response = bedrock.invoke_model(
        modelId=MODEL_ID,
        body=json.dumps(body),
        contentType='application/json',
        accept='application/json'
    )
    result = json.loads(response['body'].read())
    return {
        'statusCode': 200,
        'body': json.dumps({'result': result['choices'][0]['message']['content'], 'data': {}})
    }
```

## Web App

Built with React + Vite. Runs in any modern browser including mobile Safari and Chrome.

### Features

| Feature | Description |
|---|---|
| Auth | Cognito sign-in/up using Amazon Cognito Identity JS; JWT stored in memory (not localStorage) |
| Chat interface | Text input + mic button; sends requests to `POST /chat` REST API → Bedrock Agent |
| OI relay | Persistent WebSocket to API Gateway WebSocket API; relays Lambda task pushes to local `interpreter.server()` at `localhost:8000` |
| Vision feed | Phone camera capture via `getUserMedia`; sends frames to vision Lambda; reads responses aloud via `SpeechSynthesis` |
| File browser | Lists Box files via `box-read` Lambda; upload and share via `box-write` / `box-share` |

### Open Interpreter WebSocket Relay

This is the most critical piece of the web app for the automation feature. The browser acts as a relay between Lambda (in the cloud) and the user's local Open Interpreter server.

```typescript
// web/src/relay/OIRelay.ts
const WS_ENDPOINT = import.meta.env.VITE_WS_API_URL;   // from Dev 3
const OI_LOCAL_URL = 'http://localhost:8000/chat';

export function startRelay(userId: string, idToken: string) {
  const ws = new WebSocket(`${WS_ENDPOINT}?token=${idToken}`);

  ws.onmessage = async (event) => {
    const { task, sessionId } = JSON.parse(event.data);

    // Forward task to local Open Interpreter server
    const response = await fetch(OI_LOCAL_URL, {
      method: 'GET',
      headers: { 'Content-Type': 'application/json' },
    });

    // Stream OI output back to Lambda via WebSocket
    const reader = response.body!.getReader();
    const decoder = new TextDecoder();
    while (true) {
      const { done, value } = await reader.read();
      if (done) break;
      ws.send(JSON.stringify({ type: 'output', sessionId, data: decoder.decode(value) }));
    }
    ws.send(JSON.stringify({ type: 'done', sessionId }));
  };

  ws.onerror = (e) => console.error('OI relay error', e);
  return ws;
}
```

The relay starts automatically after the user signs in and the WebSocket connects. If the local Open Interpreter server is not running, task dispatch will fail gracefully (Dev 2's `dispatch.py` returns 503).

### Phone Camera + Vision

```typescript
// web/src/vision/CameraFeed.tsx
import { useRef, useEffect } from 'react';

export function CameraFeed({ onFrame }: { onFrame: (blob: Blob) => void }) {
  const videoRef = useRef<HTMLVideoElement>(null);

  useEffect(() => {
    navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment' } })
      .then(stream => { if (videoRef.current) videoRef.current.srcObject = stream; });
  }, []);

  const captureFrame = () => {
    const canvas = document.createElement('canvas');
    canvas.width = 512; canvas.height = 512;
    canvas.getContext('2d')!.drawImage(videoRef.current!, 0, 0, 512, 512);
    canvas.toBlob(blob => blob && onFrame(blob), 'image/jpeg', 0.85);
  };

  return (
    <div>
      <video ref={videoRef} autoPlay playsInline muted style={{ width: '100%' }} />
      <button onClick={captureFrame}>Analyze Scene</button>
    </div>
  );
}
```

`facingMode: 'environment'` uses the rear camera on mobile by default.

Frames are uploaded to a pre-signed S3 URL (requested from a lightweight upload Lambda), then the S3 key is sent to the vision router Lambda.
